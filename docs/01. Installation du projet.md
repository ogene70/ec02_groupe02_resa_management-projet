Ce guide est conçu pour vous permettre de lancer le projet Résa en quelques minutes, que vous soyez développeur Front, Back, ou nouvel arrivant sur le projet.

# 📦 11.1. Récupération du Projet et Dépendances

Commencez par récupérer le code source sur votre machine locale et installer les paquets nécessaires.

### 1. Cloner le projet depuis GitHub/GitLab
git clone https://github.com/ogene70/ec02_groupe02_resa_management-projet.git
cd ec02_groupe02_resa_management-projet

### 2. Installer toutes les dépendances Node.js

`npm install`

# 🔐 11.2. Configuration de l'Environnement

Le projet utilise des variables d'environnement pour sécuriser les accès (Base de données, Clés secrètes).

Dupliquez le fichier .env.example présent à la racine et renommez-le en .env.

Assurez-vous que le contenu de votre fichier .env correspond à ceci :

# URL de connexion à notre instance PostgreSQL Docker locale
`DATABASE_URL="postgresql://resa_user:resa_password@localhost:5432/resa_db?schema=public"
`
# Clé de chiffrement pour nos tokens (à ne jamais versionner en production !)
`JWT_SECRET="notre_super_secret_complexe_en_production_123!?"
JWT_EXPIRATION="1d"`


# 🐳 11.3. Lancement de la Base de Données (Docker)

Assurez-vous que Docker Desktop est bien allumé sur votre machine. Nous allons monter une base PostgreSQL et un client pgAdmin d'un seul coup.

# Lancer les conteneurs en arrière-plan
`docker-compose up -d`


💡 Astuce : Vous pouvez accéder à la base de données visuellement via pgAdmin sur http://localhost:5050 (Identifiants : admin@resa.com / admin).

# ⚙️ 11.4. Synchronisation Prisma et Lancement de l'API

Il ne reste plus qu'à générer le client ORM, pousser notre structure de tables vers PostgreSQL, la remplir avec quelques fausses données (Seed), et allumer le serveur !

### 1. Générer le client Prisma et pousser le schéma vers la DB
`npx prisma generate
npx prisma db push`

### 2. Peupler la base de données avec des données de test (1 Tenant, 1 User, 1 Logement)
`npx prisma db seed`

### 3. Allumer le serveur NestJS en mode développement
`npm run start:dev`

✅ Bravo !

L'application est en ligne. * API Root : http://localhost:3000/api/v1

Documentation Swagger : http://localhost:3000/api
